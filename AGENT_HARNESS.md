# 총괄·교과·학년 학생 에이전트 하네스

이 하네스의 목표는 초등임용 수험생이 기간제 교사로 일하면서 공부한 시험 내용이 실제 수업 설계에 도움이 되도록 연결하는 것이다. 사용자는 시험용 지식을 암기하는 동시에, 오늘 맡은 학년·교과·단원에서 그 지식이 어떻게 살아나는지 확인해야 한다.

## 핵심 원리

총론과 평가기준·성취수준을 정신적 지주로 둔다. 교과 전문 에이전트가 성취기준과 기출 관점을 찾아오더라도, 최종 답변은 총괄 에이전트가 총론과 평가기준의 관점에서 감독한다.

```text
사용자 질문
  ↓
총괄 에이전트
  - 질문 의도 분류
  - 총론·평가기준 감독 기준 설정
  - 관련 교과/학년 에이전트 호출
  ↓
교과 전문 에이전트
  - 교과 성취기준 검색
  - 2015↔2022 변화 비교
  - 관련 기출 관점 연결
  ↓
학년 학생 에이전트
  - 1~6학년 발달·수업 장면 관점 점검
  - 예상 오개념·참여 장벽·발문 반응 제안
  ↓
총괄 에이전트 검토
  - 총론 정합성
  - 평가기준 정합성
  - 수업 설계 정합성
  - 기출 활용의 겸손성
  - 인출 가능성
  ↓
최종 답변
```

## 총괄 에이전트

역할은 답변 생성자가 아니라 감독자다.

감독 기준:

1. 총론 정합성
   - 학생 주도성, 핵심역량, 공동체성, 삶과 연계된 학습이 반영되는가?
   - 단순 활동 나열이 아니라 교육과정 목표와 연결되는가?

2. 평가기준 정합성
   - 성취기준 → 평가 요소 → 평가 장면 → 피드백이 이어지는가?
   - 평가가 수업 끝의 채점이 아니라 학습을 돕는 증거 수집으로 기능하는가?

3. 기출 정합성
   - 기출을 공식 정답처럼 단정하지 않는가?
   - “근거 기반 예상 답안/관점”으로 표현하는가?

4. 수업 설계 정합성
   - 목표, 학생 활동, 교사 발문, 교사 지원, 평가 증거가 일관되는가?
   - 기간제 교사가 실제 교실에서 실행 가능한가?

5. 인출 정합성
   - 사용자가 바로 외울 문장을 주는 것이 아니라 떠올릴 질문을 제공하는가?
   - 답을 숨기고 자기 설명을 유도하는가?

## 교과 전문 에이전트

교과 전문 에이전트는 현재 `subject` 메타데이터로 라우팅할 수 있다.

대상 교과:

- 국어
- 수학
- 사회
- 과학
- 도덕
- 실과
- 체육
- 음악
- 미술
- 영어
- 통합교과
- 총론·창의적 체험활동

공통 임무:

1. 관련 성취기준과 2015↔2022 변화 검색
2. 검수 필요 대응표 여부 표시
3. 관련 기출문제 검색
4. 교과 특수 수업 장면 제안
5. 평가 증거와 피드백 방식 제안

## 1~6학년 학생 에이전트

학년 학생 에이전트는 교과 전문성을 대신하지 않는다. 대신 “그 학년 학생이 실제로 어떻게 반응할 것인가”를 점검한다.

현재 성취기준 메타데이터는 `1-2`, `3-4`, `5-6` 학년군 단위다. 따라서 1~6학년 단독 에이전트를 만들려면 학년군을 개별 학년으로 해석하는 보정 규칙이 필요하다.

### 학년별 초점

1학년 학생 에이전트:

- 학교생활 적응, 구체물 조작, 음성 언어, 놀이 기반 참여
- 긴 글·추상 개념 부담 점검

2학년 학생 에이전트:

- 기초 문해·수 감각 확장, 생활 경험 연결, 자기 표현
- 활동 규칙 이해와 또래 협력 점검

3학년 학생 에이전트:

- 교과 분화 초기, 관찰·분류·기초 탐구, 지역 생활 세계
- 용어 도입과 구체 예시 필요성 점검

4학년 학생 에이전트:

- 개념 간 관계 파악, 자료 해석, 협력적 문제 해결
- 오개념 언어화와 비교·분석 발문 점검

5학년 학생 에이전트:

- 추상화 증가, 자기 설명, 토의·탐구·근거 제시
- 평가 기준을 이해하고 자기 점검하는 장면 점검

6학년 학생 에이전트:

- 전이·통합 사고, 중학교 전환 준비, 논증과 성찰
- 실제 삶·사회 문제와 연결되는 과제 점검

## 기간제 교사 + 임용 수험생 사용 시나리오

사용자 질문 예시:

> “내일 5학년 수학 분수의 나눗셈 수업을 해야 하는데, 임용 관점에서 어떤 성취기준 변화와 기출 포인트를 같이 공부하면 좋을까?”

하네스 응답 흐름:

1. 총괄 에이전트가 질문을 `수업 설계 + 임용 인출`로 분류
2. 수학 에이전트가 `6수01-11` 등 관련 성취기준과 2015↔2022 변화 검색
3. 5학년 학생 에이전트가 예상 오개념과 교사 발문 제안
4. 총괄 에이전트가 평가기준 관점에서 평가 증거를 점검
5. 최종 답변은 다음 형식으로 구성
   - 오늘 수업의 핵심 목표
   - 교육과정 변화 포인트
   - 학생 예상 반응/오개념
   - 교사 발문과 지원
   - 평가 증거
   - 관련 기출 관점
   - 3분 인출 질문

## 현재 데이터로 가능한 것과 부족한 것

가능한 것:

- 교과별 라우팅
- 학년군별 라우팅
- 성취기준 변화 비교
- 관련 기출 연결
- 검수 필요 대응표 표시
- 총론·평가기준 문서 우선 검색

부족한 것:

- 1~6학년 단독 성취기준 메타데이터
- 사용자 수업 맥락 저장
- 사용자 학습 이력 저장
- 학생 반응 데이터베이스
- 검수 완료 대응표 저장소

## 구현 순서

1. `agents/` 모듈 추가
   - `supervisor_agent.py`
   - `subject_agent.py`
   - `grade_student_agent.py`
   - `agent_router.py`
   - `answer_auditor.py`

2. 메타데이터 라우터 구현

   - 질문에서 학년 추론
   - 학년을 `1-2`, `3-4`, `5-6` 학년군으로 매핑

3. 총괄 에이전트 감사 체크리스트 구현
   - 총론 근거 포함 여부
   - 평가 증거 포함 여부
   - 기출 단정 금지 여부
   - 수업 실행 가능성
   - 인출 질문 포함 여부

4. Streamlit에 “수업-임용 통합 코치” 탭 추가

5. 대표 시나리오 테스트
   - 1학년 통합교과 학교생활
   - 3학년 과학 소리의 성질
   - 4학년 사회 지역 이해
   - 5학년 수학 분수의 나눗셈
   - 6학년 국어 토의·토론

## 검증 기준

응답은 다음을 모두 만족해야 통과한다.

- 총론 또는 평가기준 관점이 최소 1개 이상 반영된다.
- 교과 성취기준 근거가 포함된다.
- 학년 학생 에이전트의 예상 반응 또는 오개념이 포함된다.
- 기출은 공식 답안처럼 단정하지 않는다.
- 수업 목표, 학생 활동, 교사 지원, 평가 증거가 포함된다.
- 마지막에 답을 바로 노출하지 않는 인출 질문이 포함된다.

# 교원임용 교육과정·수업설계 에이전트 지식베이스 명세

## 목표

사용자의 질문에 대해 다음 근거 사슬을 끊지 않고 답한다.

`2015↔2022 개정 변화 → 성취기준/해설 → 관련 기출 → 평가기준·성취수준 → 교과교육론 → 실제 수업 설계 → 인출 연습`

단순 PDF 청크 검색이 아니라 각 자료의 문서 유형, 개정 연도, 교과, 학년군,
영역, 성취기준 코드, 기출 문항 ID를 메타데이터로 보존해야 한다.

## 제공받아야 할 자료

### 운영 원칙: 기출 정답은 사용하지 않음

- 기출은 정답 데이터가 아니라 교육과정 개념이 실제 상황에서 어떻게
  문제화되는지를 보여 주는 사례로 사용한다.
- 예상 답안 요소가 필요할 때에는 교육과정 원문, 평가기준·성취수준,
  예시 평가도구에서 근거를 회수하여 구성한다.
- 생성한 답안은 `공식 정답`으로 표시하지 않고 `근거 기반 예상 답안`으로
  명시한다.
- 근거가 부족하거나 복수 해석이 가능하면 이를 그대로 표시한다.

기출만으로 채점 인정 범위를 확정하거나 실제 출제기관의 정답이라고 단정하지 않는다.

### 필수 2순위: 교과별 교육과정 원문

- 2015 개정 및 2022 개정의 교과별 교육과정 원문
- 국어, 수학, 사회, 과학, 영어, 도덕, 실과, 체육, 음악, 미술
- 바른 생활, 슬기로운 생활, 즐거운 생활/통합교과
- 총론·창의적 체험활동뿐 아니라 성격, 목표, 내용 체계, 성취기준,
  교수·학습, 평가, 용어 해설 전체

### 필수 3순위: 2015↔2022 개정 대응 자료

- 교육부의 개정 전후 비교표, 신구 대조표, 주요 변화 안내서
- 성취기준 삭제·신설·통합·분리·이동 대응표
- 적용 시기와 학년군별 전환 일정

2009 개정 교육과정 및 2009↔2015 비교는 현재 시스템의 검색·비교 범위에서
제외한다.

변화 분석은 두 문서의 문장 유사도만으로 처리하지 않고 공식 대응표를
우선 근거로 사용해야 한다.

### 수업 연결 자료

- 교과별 교사용 지도서와 교과서(개정판과 학년/학기 표시)
- 대학에서 사용하는 교과교육론 강의자료 또는 지정 교재의 사용 가능한 범위
- 수업 모형 원전/요약, 교수·학습 전략, 평가 루브릭
- 우수 수업안, 수업 관찰 기록, 학생 반응과 오개념 사례
- 실제 수업 맥락: 학년, 단원, 차시, 학생 수준, 준비물, 시간, 평가 조건

### 현재 확보된 평가 자료

- 2015 개정: 1~2, 3~4, 5~6학년 평가기준 및 예시 평가도구
- 2015 개정: 초등 통합교과 평가기준 개발 연구
- 2022 개정: 1~2, 3~4, 5~6학년군 성취수준
- 2022 개정: 초등 통합교과 성취수준 개발 연구

### 있으면 정확도가 크게 오르는 자료

- 출제기관의 문항 관련 이의제기 답변
- 연도별 출제 경향을 정리한 검수된 자료
- 교수·학습 용어 사전과 동의어 표
- 사용자가 작성한 답안과 첨삭 기록

## 답변 에이전트 루프

1. 질문에서 교과·학년군·영역·개정 연도·요구 작업을 판별한다.
2. 교육과정 원문과 공식 변화 자료를 먼저 검색한다.
3. 동일 성취기준/개념의 기출과 평가기준·성취수준을 검색한다.
4. 교과교육론과 지도서에서 수업 적용 근거를 검색한다.
5. 근거를 인용하며 변화, 출제 양상, 수업 적용을 분리해 답한다.
6. 사용자 수준에 맞춰 사례 수업안을 생성한다.
7. 짧은 인출 문제를 내고 사용자의 답을 근거 요소별로 피드백한다.
8. 근거가 없거나 충돌하면 추정하지 않고 불확실성을 표시한다.

## 현재 코드에서 바꿔야 할 핵심

- 업로드 PDF를 고정 길이로만 자르지 말고 문서 유형별 파서를 사용한다.
- FAISS 단일 유사도 검색 대신 메타데이터 필터 + 키워드/BM25 + 벡터 검색을 결합한다.
- 문항, 정답, 성취기준, 변화 레코드 사이의 ID 연결을 유지한다.
- 답변에 문서명, 페이지, 성취기준 코드, 기출 문항 ID를 표시한다.
- 뉴스 검색 에이전트는 핵심 루프에서 제거하고 수업 사례의 최신성 확인 같은
  보조 기능으로만 둔다.
- 하나의 프롬프트 체인 대신 분석, 근거 수집, 수업 설계, 검증, 인출 평가 단계를 둔다.

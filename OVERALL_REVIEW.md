# 초등임용 교육과정·수업설계 코치 총괄 검토

검토일: 2026-06-23

## 1. 현재 결론

현재 프로젝트는 다음의 핵심 줄기를 갖춘 상태다.

사용자 질문 → 교과·학년 라우팅 → 총론·평가기준 감독 → 교과 성취기준 비교 → 관련 기출 연결 → 학년별 학생 관점 → 수업 설계·평가 증거·인출 질문 생성 → 학습 기록 저장 → 이동평균/취약 성취기준/에빙하우스 복습 루프

따라서 “2015↔2022 교육과정 비교 도구”를 넘어, 초등임용 수험생과 기간제 교사가 시험 공부를 실제 수업 설계와 연결하는 코치 앱의 기반은 성립했다.

## 4. 구현된 주요 모듈

### 교육과정·기출 파이프라인

- `ocr_service.py`: PDF 텍스트 추출/OCR
- `batch_ocr.py`: 폴더 단위 OCR
- `curriculum_mapper.py`: 2015/2022 성취기준 추출·대응표 생성
- `comparison_service.py`: 성취기준 변화 검색·관련 기출 연결
- `rebuild_vector_db.py`: 교육과정/기출 벡터DB 재구축

### 총괄/전문 에이전트

- `agents/supervisor_agent.py`: 총론·평가기준 기반 총괄 감독
- `agents/subject_agent.py`: 교과 성취기준 변화·기출 관점 검색
- `agents/grade_student_agent.py`: 1~6학년 학생 반응/오개념/지원 관점
- `agents/agent_router.py`: 교과·학년·의도 라우팅

### 학습 루프

- `learning_db.py`: SQLite 학습 기록 DB
- `learning_service.py`: 풀이 기록, 일/주/월 이동평균, 취약 성취기준, 복습 일정, 변형 문제 생성
- `main.py`: FastAPI 엔드포인트
- `app.py`: Streamlit 대시보드

## 5. 이번 총괄 검토에서 보강한 점

학습 루프가 “기록→조회”에 머물지 않도록 다음을 추가했다.

1. 주간/월간 집계 API
   - `GET /learning/metrics/aggregate`
   - 일간 정답률뿐 아니라 주간·월간 장기 흐름과 이동평균을 확인할 수 있다.

2. 복습 완료 처리 API
   - `POST /learning/review-complete`
   - 오늘까지 도래한 특정 성취기준 복습 일정을 완료 처리한다.

3. Streamlit 학습 대시보드 보강
   - 일간 그래프 + 주간/월간 그래프
   - 오늘의 복습 항목에서 “복습 완료 처리” 버튼

4. 학습 하네스 보강
   - aggregate metrics 검증
   - review complete 검증

## 6. 현재 남은 리스크

### A. 자동 대응표 검수

389개 자동 추정 항목은 아직 사람 검수가 끝난 것이 아니다.
특히 high/medium 48개는 먼저 검수 UI를 붙이는 것이 좋다.

필요 기능:

- 검수 필요 배지
- 우선순위 필터
- 2015/2022 좌우 비교
- “맞음/수정 필요/대응 아님/보류” 저장

### B. 2015 기출 OCR 정밀도

2015 A/B는 검색용으로 복구됐지만, 정밀 인용에는 부족하다.

필요 기능:

- 문항별 OCR 수정 화면
- 원문 이미지/텍스트 병렬 검토
- 수정본을 벡터DB에 재반영

### C. 답변 품질 튜닝

현재 구조는 답변을 생성하지만, 수험생에게 정말 유용하려면 다음 관점의 튜닝이 필요하다.

- 원문 나열보다 “임용에서 왜 중요한가”를 먼저 말하기
- 수업 목표·활동·평가 증거 연결 강화
- 관련 기출은 단정하지 않고 “출제 관점”으로 제시
- 인출 질문은 답을 숨기고 사고 과정을 묻기

### D. 변형 문제 생성 고도화

현재 변형 문제는 로컬 템플릿 기반이다.
다음 단계에서는 다음 자료를 함께 검색해 LLM 생성으로 고도화할 수 있다.

- 취약 성취기준 원문
- 2015↔2022 변화 양상
- 관련 기출 chunk
- 총론/평가기준 근거
- 학년 학생 에이전트의 예상 오개념

## 7. 다음 우선순위

5. 답변 품질 프롬프트 튜닝

## 8. 판정

현재 상태는 “프로토타입 뼈대”가 아니라 “작동하는 로컬 MVP”에 가깝다.

아직 공식 서비스 수준의 검수·정밀 OCR·답변 품질 튜닝은 남아 있지만, 초등임용 수험생/기간제 교사가 매일 공부하면서 수업 설계까지 연결하는 핵심 하네스는 살아 있다.
